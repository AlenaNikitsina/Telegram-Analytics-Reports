# Telegram Analytics Reports (Lenta & App)

Проект представляет собой систему автоматической ежедневной отправки аналитических сводок в Telegram. Два независимых Python-скрипта (`nikitsina_bot_report_lenta.py` и `nikitsina_bot_app_report.py`) подключаются к ClickHouse, рассчитывают метрики за предыдущие дни, формируют текстовые отчеты и графики, и отправляют их в указанный Telegram-чат с помощью бота. Оркестрация выполняется Apache Airflow.

## 🎯 Цели проекта

- **Автоматизация рутинной отчетности:** Исключить необходимость ручного сбора метрик каждое утро.
- **Визуализация динамики:** Предоставлять не только цифры за вчера, но и графики изменений за последние 7 дней.
- **Два уровня детализации:**
    - Отчет по ленте новостей (ключевые метрики вовлеченности).
    - Сводный отчет по всему приложению (кросс-функциональная аналитика).
- **Регулярность:** Доставка отчетов строго по расписанию в 11:00 ежедневно.

## 📊 Источники данных

Оба отчета получают данные из таблиц **ClickHouse**, которые содержат агрегированную информацию о событиях пользователей:
- **`feed_actions`** (действия в ленте: просмотры, лайки, показы)
- **`message_actions`** (действия в мессенджере: отправка/получение сообщений)
- Справочная информация о пользователях (дата регистрации, платформа, источник трафика)

## 🤖 Отчет №1: Лента новостей (`nikitsina_bot_report_lenta.py`)

Этот отчет фокусируется исключительно на активности пользователей в ленте.

### Состав отчета:
1.  **Текстовая часть (за предыдущий день):**
    - `DAU` (Daily Active Users) — уникальные пользователи ленты.
    - `Просмотры` (Views) — общее количество просмотров контента.
    - `Лайки` (Likes) — общее количество лайков.
    - `CTR` (Click-Through Rate) — отношение лайков к просмотрам (в процентах).
2.  **Графическая часть (динамика за последние 7 дней):**
    - Линейный график с линиями для DAU, Просмотров, Лайков и CTR.

## 📱 Отчет №2: Сводка по приложению (`nikitsina_bot_app_report.py`)

Комплексный отчет, объединяющий данные ленты и мессенджера для понимания общего состояния продукта.

### Состав отчета:

**Текстовая часть (за предыдущий день):**

*   **Общая статистика:**
    *   Уникальные пользователи (общие по приложению).
    *   Уникальные пользователи по платформе: iOS и Android.
    *   Новые пользователи (впервые зарегистрировавшиеся).
    *   Новые пользователи по источнику: реклама (`ad`) и органический трафик (`organic`).
*   **Сценарии использования:**
    *   Только лента новостей.
    *   Только мессенджер.
    *   Лента новостей + мессенджер.
*   **Метрики по продуктам:**
    *   *По ленте новостей:* DAU, Просмотры, Лайки, CTR.
    *   *По мессенджеру:* DAU, Количество отправленных сообщений.

**Графическая часть (2 графика за 7 дней):**

1.  **График 1 (Сценарии использования):**
    *   Уникальные пользователи (общие)
    *   Новые пользователи
    *   Только лента
    *   Только мессенджер
    *   Лента + мессенджер
2.  **График 2 (Сравнение продуктов):**
    *   *Лента:* DAU, Просмотры, Лайки, CTR
    *   *Мессенджер:* DAU, Количество сообщений

## 🔧 Используемые технологии и библиотеки

- **Язык:** Python 3.8+
- **Оркестрация:** Apache Airflow (DAG для ежедневного запуска в 11:00)
- **База данных:** ClickHouse
- **Библиотеки:**
    - `clickhouse-driver` / `pandas` — для подключения к БД и выгрузки данных.
    - `matplotlib` / `seaborn` — для построения графиков.
    - `python-telegram-bot`  — для отправки сообщений через Telegram Bot API.

## 🚀 Результат

Каждое утро в 11:00 в Telegram-чат приходят два структурированных сообщения:

1.  **Отчет по ленте** — для быстрой оценки состояния основного функционала.
2.  **Сводный отчет по приложению** — для глубокого понимания поведения пользователей, распределения по платформам и источникам трафика.

Это позволяет команде всегда быть в курсе ключевых метрик продукта без необходимости заходить в BI-системы.
